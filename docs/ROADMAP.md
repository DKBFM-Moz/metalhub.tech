# 🗺 ROADMAP — AI-калькулятор металлообработки (B2B SaaS)
> Один разработчик · 10–20 часов/неделю · Vibe coding (AI-assisted)
> Последнее обновление: 17 апреля 2026

---

## 📌 Концепция (зафиксировано)

- **Продукт:** Калькулятор встроен в сайт. Пользователь загружает чертёж → AI читает его как опытный инженер-технолог → выдаёт расчёт с открытыми ползунками → пользователь при желании корректирует вручную
- **Аудитория:** B2B SaaS — несколько заводов/компаний, каждая со своим аккаунтом и историей
- **Параметры расчёта:** единые для всех (рыночные ставки), но каждый пользователь может подкрутить ползунки под себя
- **Пересчёт:** AI один раз при загрузке чертежа → ползунки пересчитываются формулой мгновенно → кнопка "Пересчитать через AI" если нужна полная переоценка
- **Скрытых параметров нет** — все видят всё, всё прозрачно
- **Монетизация:** кредитная модель на уровне организации. Клиенты платят рублями (ЮКасса/СБП) → ты пополняешь Anthropic API отдельно (Wise/Revolut)

### Этап 2 — Формула пересчёта и ползунки
**Цель:** После AI-расчёта пользователь видит ползунки и может менять их — пересчёт идёт мгновенно формулой

**Ползунки (все открыты пользователю):**
- Материал (выбор из списка)
- Количество деталей
- Срочность (стандарт / срочно)
- Чей материал: наш / давальческий
- Тип обработки (из JSON, редактируемо)
- Машинное время — часы (из JSON, редактируемо)
- Ставка станка (рыночная по умолчанию, редактируемо)
- Коэффициент сложности (из JSON, редактируемо)

**Задачи:**
- [ ] Финализировать список ползунков вместе с инженерами
- [ ] Прописать формулу пересчёта: изменение ползунка → новая цена
- [ ] Реализовать cost engine в Worker (функция: ползунки → цена)
- [ ] Реализовать кнопку "Пересчитать через AI" — повторный запрос с учётом изменений
- [ ] Протестировать все комбинации: материал × сложность × срочность

### Этап 3 — Мультитенантная БД и кредиты
**Цель:** Организации регистрируются, получают кредиты, кредиты списываются при расчётах

**Структура D1:**
- `organizations` — компания/завод (tenant)
- `users` — сотрудники организации
- `credits` — баланс кредитов на уровне организации
- `calculations` — история расчётов (привязана к организации и пользователю)

**Задачи:**

- [ ] Реализовать приглашение сотрудников в организацию
- [ ] Начисление 3 бесплатных кредитов при регистрации
- [ ] Списание кредита при расчёте (1–3 кредита в зависимости от сложности)
- [ ] История расчётов организации

**Чекпоинт ✅:** Новая организация регистрируется → получает 3 кредита → делает расчёт → кредит списывается → расчёт в истории
